## Perfection pass updates (2026-05-16)

### ch-09 and ch-15: Docker availability check fix

Replaced `DockerClientFactory.instance().client()` (which throws on socket resolution failure and required a try/catch) with `DockerClientFactory.instance().isDockerAvailable()` in both `Saml2KeycloakIT` and `MockOAuth2ServerIT`. The new call resolves the socket correctly before attempting a client connection and returns a plain boolean, eliminating the brittle exception-based skip pattern. The `Assumptions.assumeTrue` call now sits directly on the boolean result with a precise skip message ("Docker not available (socket not reachable)"). Both `mvn verify` runs confirm BUILD SUCCESS. The container ITs remain skipped on this machine because Docker Desktop's daemon socket returns HTTP 400 (degraded state) and `/var/run/docker.sock` is absent; `isDockerAvailable()` correctly returns false in this configuration. No silent skip: the assumption message is visible in Surefire output.

### ch-12: Tautological test removed

Removed `wireMockAs_tokenEndpoint_returnsValidToken` from `OAuth2AuthorizationServerIT`. That test called `RestTemplate` directly against the WireMock stub it had just set up, exercising only WireMock's own response machinery with no Spring Security or OAuth2 client stack involvement. The two remaining tests cover the meaningful contract: `clientCredentials_fullCycle_returnsRecommendations` (full `DefaultOAuth2AuthorizedClientManager` token-fetch-and-catalog-call cycle via `RecommendationController`) and `unauthenticated_redirectsToOAuth2Login` (SecurityFilterChain redirect assertion). `mvn verify` confirms BUILD SUCCESS with Tests run: 2 for `OAuth2AuthorizationServerIT` (total module count 13, down from 14). Unused imports (`RestTemplate`, `HttpHeaders`, `HttpMethod`, `HttpStatus`, `MediaType`, `ResponseEntity`, `LinkedMultiValueMap`, `MultiValueMap`, `HttpEntity`, `URI`, `CatalogMovie`, `BeforeAll`) removed from the IT file.
